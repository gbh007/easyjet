# ✈️ EasyJet

⚠️ **Внимание:** часть кода написана с помощью ИИ

CD система, с упрощенными способами доставки кода на "прод" для домашних серверов и пет проектов.

Ключевые цели:

1. Запускатся на маломощных системах - например на RassberyPI 3
2. Настраиватся отдельно от кодовой базы приложения, для обеспечения 12 factor application
3. Запускать типовые проекты в пару кликов
4. Не ограничеватся запуском проекта и предоставить возможность для настройки произвольных скриптов - например для бекапов баз пет проектов
5. Обеспечить минимальную наблюдаемость и алерты из коробки без излишеств enterprise

Разработка будет происходить в несколько этапов:

- MVP - ключевые фичи/особености
  - Сборка docker образов и запуск docker compose
  - Приложение выполняет команды через shell (bash/sh)
  - Приложение запускается как linux daemon
  - Базовая обвязка для информации о проекте который собирается (название, скрипт, ссылка на репозиторий и т.п.)
- Этап 2 - ключевые фичи/особености
  - Метрики и алерты
  - Опция шифрования файлов
  - Создание docker compose конфигураций
  - Настройка типовых деплоев полностью из приложения
  - 2 linux daemon для самообновлений
- Этап 3+ - ключевые фичи/особености
  - Работа с git, docker напрямую
  - Поддержка агентного режима
  - Возможность разворачивания приложения в docker контейнере
  - Шаблонизатор

## Архитектура проекта

Фронт и возможно в будущем остальное пишется с использованием ИИ, просто потому что 🥴  
Выбрана hexagonal architecture в трактовке deepseek, просто так, для разнообразия 🤷‍♂️

```plain
myproject/
├── cmd/
│   └── myapp/
│       └── main.go                 # Точка входа, инициализация зависимостей
│
├── internal/
│   ├── core/                        # Ядро приложения (Domain)
│   │   ├── entity/                   # Бизнес-сущности
│   │   │   ├── user.go
│   │   │   └── order.go
│   │   ├── port/                      # Интерфейсы (порты)
│   │   │   ├── repository.go          # Исходящие порты (для БД)
│   │   │   └── service.go             # Входящие порты (Use Cases)
│   │   └── service/                   # Реализация бизнес-логики (Use Cases)
│   │       ├── user_service.go
│   │       └── order_service.go
│   │
│   └── adapter/                      # Адаптеры (реализации портов)
│       ├── repository/                 # Исходящие адаптеры (БД, API)
│       │   ├── postgres/
│       │   │   ├── user_repo.go        # Реализация repository порта для Postgres
│       │   │   └── order_repo.go
│       │   └── memory/
│       │       └── user_repo.go        # In-memory реализация для тестов
│       └── handler/                    # Входящие адаптеры (доставка данных)
│           ├── http/
│           │   ├── handler.go           # HTTP handlers
│           │   └── dto.go               # Data Transfer Objects
│           ├── grpc/
│           │   └── server.go
│           └── consumer/                 # Consumer'ы для очередей
│               └── kafka_consumer.go
│
├── pkg/                               # Публичные библиотеки
│   └── utils/                         # Вспомогательные утилиты
│
├── config/                            # Конфигурационные файлы
│   └── config.go
│
├── migrations/                         # Миграции БД
│   └── ...
│
├── go.mod
└── go.sum
```
