# DFD Диаграммы (Data Flow Diagram)

Данный документ описывает потоки данных в системе EasyJet через DFD-диаграммы.

## 1. Регистрация нового проекта

**Описание:** Процесс создания нового проекта в системе. Пользователь отправляет конфигурацию проекта, система сохраняет её в базу данных, создаёт директорию для проекта и при необходимости инициализирует Git-репозиторий.

```mermaid
flowchart TD
    User[Пользователь]
    CreateProject([1.0 Создание проекта])
    ManageFS([2.0 Управление файловой системой])
    GitOps([3.0 Git операции])
    DB1[(D1: База данных проектов)]
    FS1[(D2: Файловое хранилище)]

    User -->|Отправляет конфигурацию проекта| CreateProject
    CreateProject -->|Сохраняет данные проекта| DB1
    CreateProject -->|Запрашивает создание директории| ManageFS
    ManageFS -->|Создаёт директорию проекта| FS1
    ManageFS -->|Возвращает путь к директории| CreateProject
    CreateProject -->|Проверяет наличие Git URL| GitOps
    GitOps -->|Инициализирует репозиторий| FS1
    GitOps -->|Выполняет git pull| FS1
    CreateProject -->|Возвращает ID проекта| User
```

## 2. Запуск развёртывания проекта

**Описание:** Пользователь инициирует запуск развёртывания проекта. Система создаёт запись о запуске в базе данных и помечает её как ожидающую выполнения.

```mermaid
flowchart TD
    User[Пользователь]
    StartRun([1.0 Запуск проекта])
    DB1[(D1: База данных проектов)]

    User -->|Отправляет запрос на запуск| StartRun
    StartRun -->|Создаёт запись о запуске| DB1
    StartRun -->|Возвращает ID запуска| User
```

## 3. Обработка ожидающих развёртываний

**Описание:** Фоновый рабочий процесс периодически проверяет базу данных на наличие ожидающих развёртываний и обрабатывает их. Этот процесс выполняется автоматически без участия пользователя.

```mermaid
flowchart TD
    Handler[Планировщик задач]
    SearchTask([1.0 Поиск задач для запуска])
    HandleOneTask([2.0 Обработка задачи])
    GitHandle([3.0 Git операции])
    ShellExecute([4.0 Выполнение скриптов])
    DB1[(D1: База данных проектов)]
    FS1[(D2: Файловое хранилище)]

    SearchTask -->|Получает список ожидающих запусков| DB1
    SearchTask -->|Передаёт ID запуска| HandleOneTask
    HandleOneTask -->|Обновляет статус на Processing| DB1
    HandleOneTask -->|Получает конфигурацию проекта| DB1
    HandleOneTask -->|Проверяет наличие Git| GitHandle
    GitHandle -->|Получает список коммитов| FS1
    GitHandle -->|Сохраняет логи коммитов| DB1
    GitHandle -->|Выполняет git pull| FS1
    HandleOneTask -->|Выполняет скрипты этапов| ShellExecute
    ShellExecute -->|Запускает shell-скрипты| FS1
    ShellExecute -->|Возвращает вывод скрипта| HandleOneTask
    HandleOneTask -->|Сохраняет результаты этапов| DB1
    HandleOneTask -->|Обновляет статус на Success/Fail| DB1
```

## 4. Получение информации о проекте

**Описание:** Пользователь запрашивает информацию о конкретном проекте или список всех проектов. Система извлекает данные из базы данных и возвращает их пользователю.

```mermaid
flowchart TD
    User[Пользователь]
    GetProjects([1.0 Получение проектов])
    GetProject([2.0 Получение проекта])
    DB1[(D1: База данных проектов)]

    User -->|Запрашивает список проектов| GetProjects
    GetProjects -->|Извлекает данные проектов| DB1
    GetProjects -->|Возвращает список проектов| User
    User -->|Запрашивает проект по ID| GetProject
    GetProject -->|Извлекает данные проекта| DB1
    GetProject -->|Возвращает данные проекта| User
```

## 5. Получение истории запусков проекта

**Описание:** Пользователь запрашивает историю запусков для конкретного проекта или детали отдельного запуска. Система извлекает данные о запусках включая логи этапов и Git-логи.

```mermaid
flowchart TD
    User[Пользователь]
    GetRuns([1.0 Получение запусков])
    GetRun([2.0 Получение деталей запуска])
    DB1[(D1: База данных проектов)]

    User -->|Запрашивает историю запусков| GetRuns
    GetRuns -->|Извлекает список запусков| DB1
    GetRuns -->|Возвращает историю запусков| User
    User -->|Запрашивает детали запуска| GetRun
    GetRun -->|Извлекает данные запуска| DB1
    GetRun -->|Извлекает логи этапов| DB1
    GetRun -->|Извлекает Git-логи| DB1
    GetRun -->|Возвращает детали запуска| User
```

## 6. Обновление конфигурации проекта

**Описание:** Пользователь обновляет конфигурацию существующего проекта (Git URL, ветку, скрипты этапов). Система сохраняет изменения в базе данных.

```mermaid
flowchart TD
    User[Пользователь]
    UpdateProject([1.0 Обновление проекта])
    DB1[(D1: База данных проектов)]

    User -->|Отправляет обновлённую конфигурацию| UpdateProject
    UpdateProject -->|Проверяет существование проекта| DB1
    UpdateProject -->|Сохраняет изменения| DB1
    UpdateProject -->|Возвращает подтверждение| User
```

## Условные обозначения

| Элемент          | Нотация (объявление)            | Нотация (в потоках) | Описание                           |
| ---------------- | ------------------------------- | ------------------- | ---------------------------------- |
| Внешняя сущность | `User[Название]`                | `User`              | Пользователь или внешняя система   |
| Процесс          | `CreateProject([1.0 Название])` | `CreateProject`     | Действие или функция системы       |
| Хранилище данных | `DB1[(D1: Название)]`           | `DB1`               | База данных или файловое хранилище |
| Поток данных     | `-->\|Описание\|`               | `-->\|Описание\|`   | Направление и содержание потока    |

**Важно:** В потоках данных используйте только имена элементов без повторения нотации. Например: `User -->|Отправляет данные| CreateProject`

**Важно:** Идентификатор процесса должен быть осмысленным (`CreateProject`, `GetRuns`), а не общим (`Process1`, `Process2`).

## Хранилища данных

| ID  | Название             | Описание                                                                                |
| --- | -------------------- | --------------------------------------------------------------------------------------- |
| D1  | База данных проектов | SQLite/PostgreSQL база с таблицами: projects, stages, runs, run_stages, run_git_commits |
| D2  | Файловое хранилище   | Файловая система сервера для хранения скриптов и проектов                               |
