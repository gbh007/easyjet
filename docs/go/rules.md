# Правила при написании кода на Go

- После любой модификации Go кода форматируй его командой `task go:format`
- Написанный код ОБЯЗАТЕЛЬНО должен проходить линтер `task go:lint`
  - При наличии проблем их можно попытаться исправить `task go:lint:fix`
- Написанный код ОБЯЗАТЕЛЬНО должен проходить тесты `task go:test`
- При наличии проблем с зависимостями их можно попытаться исправить командой `task go:tidy`
- Написанный код ОБЯЗАТЕЛЬНО должен собираться `task build:server`
