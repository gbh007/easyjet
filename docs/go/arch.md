# Архитектура проекта для языка Go

Приложение использует **Hexagonal Architecture** (Ports & Adapters).
Ядро приложения изолировано от внешних зависимостей (БД, HTTP, файловая система, git).

## Структура проекта

```plain
easyjet/
├── cmd/
│   └── server/
│       └── main.go                     # Точка входа, инициализация зависимостей
├── config/                             # Конфигурация приложения (toml)
├── internal/
│   ├── core/                           # Ядро приложения (Domain)
│   │   ├── entity/                     # Бизнес-сущности
│   │   │   ├── project.go              # Сущность проекта
│   │   │   ├── project_run.go          # Сущность запуска проекта
│   │   │   ├── commit.go               # Сущность Git-коммита
│   │   │   └── event.go                # Сущности событий
│   │   ├── port/                       # Интерфейсы (порты)
│   │   │   └── interfaces.go           # Определение портов
│   │   └── service/                    # Реализация бизнес-логики (Use Cases)
│   │       ├── service.go              # Основной сервис
│   │       ├── create_project.go       # Создание проекта
│   │       ├── update_project.go       # Обновление проекта
│   │       ├── run_project.go          # Запуск проекта
│   │       └── run_rotation.go         # Ротация запусков
│   └── adapter/                        # Адаптеры (реализации портов)
│       ├── exec/                       # Исходящий порт: выполнение команд
│       │   └── shellexec/              # Реализация через shell
│       ├── filesystem/                 # Исходящий порт: работа с ФС
│       │   └── filesystem/             # Реализация работы с файловой системой
│       ├── git/                        # Исходящий порт: работа с git
│       │   └── shellgit/               # Реализация через shell git
│       ├── handler/                    # Входящие порты (доставка данных)
│       │   ├── httpapi/                # HTTP API (Echo framework)
│       │   ├── scheduler/              # Scheduler для cron-задач
│       │   └── worker/                 # Worker для фоновых задач
│       ├── pubsub/                     # Исходящий порт: публикация/подписка на события
│       │   └── eventbus/               # Реализация event bus
│       ├── repository/                 # Исходящий порт: персистентность
│       │   ├── gorm/                   # Реализация через GORM
│       │   └── postgres/               # Реализация через pgx (SQL-запросы)
│       ├── internal/                   # Внутренние утилиты и инструменты для адаптеров
│       └── scheduler/                  # Внутренний адаптер: очередь событий планировщика
├── migrations/                         # Миграции базы данных
├── frontend/                           # Vue.js фронтенд
└── docs/                               # Документация
```
